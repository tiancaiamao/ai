# Workflow Team Mode Guide

This guide documents how to run the `implement` phase using the new team-mode executor:

- Script: `scripts/implement-team.sh`
- Runtime backend: `~/.ai/skills/ag/ag`
- Plan input: `PLAN.yml` (from `plan` skill)

## What It Does

`implement-team.sh` automates the implementation phase with:

1. Team namespace setup (`ag team init/use`)
2. Plan import (`ag task import-plan`)
3. Dependency-aware scheduling (`ag task next`)
4. Parallel worker loops (`--workers N`)
5. Two-stage review per task:
   1. spec compliance
   2. code quality
6. Task completion/failure updates (`ag task done/fail`)
7. Team completion (`ag team done`)

## Prerequisites

1. `ag` binary available at `~/.ai/skills/ag/ag` (or set `AG_BIN`)
2. Prompt files exist:
   - `~/.ai/skills/implement/prompts/implementer.md`
   - `~/.ai/skills/implement/prompts/spec-reviewer.md`
   - `~/.ai/skills/implement/prompts/quality-reviewer.md`
3. A valid `PLAN.yml` generated by the `plan` skill

## Quick Start

```bash
cd <repo-root>

# example paths from workflow artifacts
PLAN=.workflow/artifacts/features/feature/PLAN.yml
SPEC=.workflow/artifacts/features/feature/SPEC.md

~/.ai/skills/workflow/scripts/implement-team.sh run \
  --plan "$PLAN" \
  --spec "$SPEC" \
  --team feat-user-auth \
  --workers 3
```

## Commands

```bash
# main execution
implement-team.sh run [options]

# inspect current state
implement-team.sh status [--team TEAM_ID]

# remove a team workspace
implement-team.sh cleanup [--team TEAM_ID] [--force]
```

## Important Options

- `--workers N`: number of parallel worker loops
- `--max-rounds N`: max repair loops per task when reviews reject
- `--implement-timeout SEC`: timeout per implementer run
- `--review-timeout SEC`: timeout per reviewer run
- `--results-dir DIR`: where per-task reports are written
- `--skip-import`: don't import PLAN tasks if you already loaded queue manually
- `--dry-run`: simulate full flow without spawning agents
- `--force-cleanup`: delete team workspace after success

## Typical Workflow Integration

1. `workflow` phase order unchanged:
   1. brainstorm
   2. spec
   3. plan
   4. implement
2. In implement phase, run `implement-team.sh run ...`.
3. After script succeeds, continue normal flow (`workflow-ctl advance`).

## Resume / Recovery

If execution was interrupted:

1. Re-run with same `--team` value.
2. Script will `ag team use <team>`.
3. If tasks already exist in queue, import is skipped automatically.
4. Remaining pending tasks continue from queue state.

## Examples

### 1) Dry-run validation

```bash
implement-team.sh run \
  --plan .workflow/artifacts/features/feature/PLAN.yml \
  --spec .workflow/artifacts/features/feature/SPEC.md \
  --team feat-dry-run \
  --dry-run
```

### 2) Recover and continue an existing team

```bash
implement-team.sh status --team feat-user-auth
implement-team.sh run \
  --plan .workflow/artifacts/features/feature/PLAN.yml \
  --spec .workflow/artifacts/features/feature/SPEC.md \
  --team feat-user-auth \
  --workers 2
```

### 3) Force cleanup

```bash
implement-team.sh cleanup --team feat-user-auth --force
```
